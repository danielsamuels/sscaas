What is SSCaaS?

It's the new "aaS" thing! Only this time we're making Slack even better! No, we didn't think that was possible either. The goal is to make it easy to extend the functionality of Slack and provide useful tools and API integrations through slash commands.

What's currently available?

At the moment SSCaaS has the following slash commands available:

  • Dictionary lookup
  • Developer Excuses
  • Reddit linker
  • Sing
  • Site availability checker
  • SoundCloud track finder
  • Troutslap
  • Urban Dictionary lookup

Great! How do I get started?

The first thing you'll need to do is create an Incoming Webhook which will allow SSCaaS to post messages into your Slack instance in response to slash commands.

You should be given a Webhook URL, keep a hold of this as you'll need it later.

The next step is to add a Slash Command. The URL and setup will vary based on which command you would like to use, click the links above for the full instructions for each command.

I don't trust you, I want to self-host this.

Go for it! You should be able to simply go get the package and install it without any issues. I use nginx and supervisor to handle the HTTP requests and maintain the process, you can find the configuration scripts inside the server-files/ directory.

Why are you using a .eu TLD?

Because it was £1.99 and this is only a hobby project (also, the .com is taken).

Where is this hosted?

It's currently hosted on a London based Digital Ocean droplet and sitting behind Cloudflare, this should allow it to scale reasonably well.
